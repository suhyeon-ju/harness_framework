# PRD: doPOST

## 목표
자영업자가 매일 Threads 게시글 작성에 쓰는 시간을 0에 가깝게 줄인다. 업종·말투만 설정하면 AI가 글을 써주고, 사용자는 복사해서 올리기만 하면 된다.

## 사용자

### MVP 타겟
- 동네 자영업자 (카페, 음식점, 부동산, 헤어샵)
- Threads를 운영하고 싶지만 매일 글 쓰기 힘든 사람
- 스마트폰으로 간단하게 쓸 수 있는 툴이 필요한 사람

### 2차 확장 타겟
- 프리랜서 개발자
- 퍼스널 브랜딩이 필요한 1인 사업자

## 플랜 구성

| 기능 | 무료 | 프로 (월 5,000원) |
|------|------|------|
| 글 생성 | 1일 1개 | 1일 3개 |
| 주제 설정 | 직접 입력만 | 자동 생성 + 직접 입력 |
| AI 보완 옵션 | ❌ | ✅ (하루 10회 제한) |
| 자동 생성 기간 설정 | ❌ | ✅ |
| 보관함 저장 | 7일만 보관 | 무제한 |
| 해시태그 자동 추가 | ❌ | ✅ |

### Claude API 비용 관리
- 글 1개 생성 ≈ 약 500~1000 토큰
- 프로 유저 1명 월 비용 ≈ 약 100~200원
- AI 보완 옵션: 하루 10회 제한 (비용 폭증 방지)

### 무료 → 유료 전환 팝업
```
🔒 유료 플랜 전용 기능이에요
해당 기능은 프로 플랜에서 지원하는 서비스입니다.
[플랜 업그레이드] [닫기]
```

## MVP 제외 사항
- Threads API 자동 업로드 (13단계, API 심사 필요)
- 인스타그램 카드뉴스 (12단계)
- 이메일 발송
- AI 이미지 생성

## 플랫폼 전략
```
1차: Threads 특화 (시장성 테스트)
2차: 인스타그램 카드뉴스 확장
3차: Threads API 연동으로 원클릭 자동 업로드
```

## 페이지 구조

### 라우팅 분기
```
비로그인 → / (랜딩 페이지)
로그인 → /dashboard 자동 이동

무료 플랜 → 대시보드 (기능 일부 제한)
프로 플랜 → 대시보드 (전체 기능)
구독 만료 → 결제 페이지 이동
```

### 1. 랜딩 페이지 (/)
- 서비스 소개
- 플랜 비교표
- 카카오 로그인 버튼
- 로그인 상태면 /dashboard 자동 이동

### 2. 대시보드 (/dashboard)

#### 상단 헤더
- 업종명 표시 (예: ☕ 카페)
- 남은 생성 횟수 (2/3)

#### 요일 탭
```
월  화  수  목  금  토  일
```
- 오늘 날짜: 오렌지 배경으로 강조
- 글 없는 날: 점 없음
- 글 생성 (미업로드): 🟠 주황 점
- 업로드 완료: 🟢 초록 점

#### 콘텐츠 영역
- 오늘 선택 시: 글 전체 내용 + 복사/수정/삭제/업로드완료 버튼
- 다른 요일 선택 시: 첫 줄만 표시 → 터치 시 전체 펼쳐짐

#### 하단
- 글 생성하기 버튼 (고정 CTA)
- 하단 탭 바: 🏠 홈 / ✍️ 생성 / 📂 보관함 / ⚙️ 설정

### 3. 글 생성 (/generate)

#### 1단계: 주제 설정
- 자동 생성 / 직접 입력 선택
- 말투 선택 (설정값 기본 적용)
- 생성하기 버튼

#### 2단계: 로딩
- "AI가 글을 쓰고 있어요 ✍️" 애니메이션

#### 3단계: 결과 (3개 카드)
- 글 전체 내용 표시
- 각 카드 버튼: 수정 / AI보완 / 저장 / 복사

#### AI 보완 옵션 (프로 플랜 / 하루 10회 제한)
```
더 유머러스하게 😄
더 짧게 ✂️
더 길게 📝
해시태그 추가 #
톤 변경 🔄
다시 생성 🔁
```

#### 4단계: 저장
- 저장 시 보관함에 추가 + 대시보드 해당 날짜 🟠 점 표시

### 4. 보관함 (/history)

#### 상단 필터 탭
```
전체 / 즐겨찾기 / 업로드완료 / 미업로드
```

#### 글 목록 (최신순)
- 카드 형태: 첫 줄 미리보기 + 날짜
- 🟠 미업로드 / 🟢 업로드완료 배지
- ♡ 즐겨찾기 버튼
- 터치 시 전체 내용 펼쳐짐 + 복사/수정/삭제 버튼

#### 플랜별 제한
- 무료: 7일치만 보관, 7일 이후 접근 시 업그레이드 팝업
- 프로: 무제한 보관

### 5. 설정 (/settings)

#### 내 계정
- 카카오 프로필 정보
- 구독 플랜 확인 + 업그레이드 버튼
- 회원 탈퇴 (탈퇴 시 데이터 전체 삭제)
- 로그아웃

#### 글쓰기 설정
- 업종 선택: 카페 / 음식점 / 부동산 / 헤어샵 / 직접 입력
- 말투/톤: 반말체 / 존댓말체 / 친근한 말투 / 정보 전달형 / 공감 유도형
- 이모지 사용: 많이 / 적당히 / 안 씀
- 해시태그 자동 추가: ON / OFF (프로 플랜)

#### 자동 생성 설정 (프로 플랜)
- 자동 생성 기간 설정 (시작일 ~ 종료일)
- 주제 방향 설정 (키워드 입력)

#### 고객센터
- 공지사항
- 문의하기 (카카오톡 채널 연결)

### 6. 온보딩 (/onboarding)
최초 로그인 시 1회 실행.
```
1. 업종 선택
2. 말투/톤 선택
3. 완료 → 대시보드 이동
```

## Claude API 프롬프트 설계

### 기본 프롬프트 구조
```
당신은 {업종} 사장님의 Threads SNS 담당자입니다.
아래 조건에 맞는 Threads 게시글 3개를 작성해주세요.

[조건]
- 업종: {업종}
- 말투: {말투}
- 이모지: {이모지 옵션}
- 해시태그: {해시태그 옵션}
- 주제: {주제}
- 글자 수: 150자 내외

[출력 형식]
글1: ...
글2: ...
글3: ...
```

### 업종별 특성
- 카페: 음료/디저트/분위기/날씨 감성 강조
- 음식점: 오늘의 메뉴/재료/영업시간 중심
- 부동산: 시세/매물/생활정보 신뢰감 있게
- 헤어샵: 스타일링/예약/이벤트 친근하게

### 말투 프롬프트 반영
- 반말체: "친한 친구에게 말하듯 반말로"
- 존댓말체: "고객에게 정중하게 존댓말로"
- 친근한 말투: "동네 단골손님에게 말하듯 편안하게"
- 정보 전달형: "핵심 정보만 간결하게"
- 공감 유도형: "공감과 질문을 유도하는 방식으로"

### 이모지 옵션 반영
- 많이: "이모지를 문장마다 적극적으로 사용"
- 적당히: "이모지를 2~3개 적절히 사용"
- 안 씀: "이모지 없이 텍스트만"

### 해시태그 옵션 반영
- ON: "마지막 줄에 관련 해시태그 3~5개 추가"
- OFF: "해시태그 없이"

### AI 보완 옵션 프롬프트
- 더 유머러스하게: "위 글을 더 유머러스하고 재치있게 수정해줘"
- 더 짧게: "위 글을 50자 이내로 핵심만 남기고 줄여줘"
- 더 길게: "위 글을 200자 이상으로 자세하게 늘려줘"
- 해시태그 추가: "위 글 마지막에 해시태그 3~5개 추가해줘"
- 톤 변경: "위 글을 {선택한 말투}로 바꿔줘"
- 다시 생성: 기본 프롬프트로 재호출
