# doPOST

## 기술 스택
- Next.js 15
- TypeScript strict mode
- Tailwind CSS
- Supabase (DB + Auth)
- Claude API (글 생성)

## 아키텍처 규칙
- CRITICAL: 모든 API 로직은 app/api/ 라우트 핸들러에서만 처리
- CRITICAL: 클라이언트 컴포넌트에서 직접 외부 API 호출 금지
- CRITICAL: Claude API 키는 서버에서만 사용 (클라이언트 노출 금지)
- CRITICAL: Supabase plan 컬럼은 서버에서만 업데이트 (클라이언트 직접 변경 금지)
- CRITICAL: Threads 액세스 토큰은 서버에서만 사용 (클라이언트 노출 금지)
- 컴포넌트는 components/ 폴더에 저장
- 타입은 types/ 폴더에 분리
- 재사용 로직은 hooks/ 폴더에 분리
- 환경변수는 .env.local 에서만 관리

## 플랜 구성 및 수익화

### 플랜 비교
| 기능 | 무료 | 프로 (월 5,000원) |
|------|------|------|
| 글 생성 | 1일 1개 | 1일 3개 |
| 주제 설정 | 직접 입력만 | 자동 생성 + 직접 입력 |
| AI 보완 옵션 | ❌ | ✅ (하루 10회 제한) |
| 자동 생성 기간 설정 | ❌ | ✅ |
| 보관함 저장 | 7일만 보관 | 무제한 |
| 해시태그 자동 추가 | ❌ | ✅ |

### Claude API 비용 관리
- 글 1개 생성 ≈ 약 500~1000 토큰
- 프로 유저 1명 월 비용 ≈ 약 100~200원
- AI 보완 옵션: 하루 10회 제한 (비용 폭증 방지)
- 구독료 5,000원 - API 비용 ≈ 순수익 4,800원/유저

### 무료 → 유료 전환 팝업
무료 플랜 사용자가 유료 기능 클릭 시 팝업 출력
```
🔒 유료 플랜 전용 기능이에요
해당 기능은 프로 플랜에서 지원하는 서비스입니다.
[플랜 업그레이드] [닫기]
```

### 2. 대시보드 (/dashboard)

#### 상단 헤더
- 업종명 표시 (예: ☕ 카페)
- 남은 생성 횟수 (2/3)

#### 요일 탭
```
월  화  수  목  금  토  일
```
- 오늘 날짜: 오렌지 배경으로 강조
- 글 없는 날: 점 없음
- 글 생성 (미업로드): 🟠 주황 점
- 업로드 완료: 🟢 초록 점
- 1차: 수동 체크로 완료 처리
- 2차: n8n/Make 연동 시 자동 처리

#### 콘텐츠 영역
- 오늘 선택 시: 글 전체 내용 + 복사/수정/삭제/업로드완료 버튼
- 다른 요일 선택 시: 첫 줄만 표시 → 터치 시 전체 펼쳐짐

#### 하단
- 글 생성하기 버튼 (고정 CTA)
- 하단 탭 바: 🏠 홈 / ✍️ 생성 / 📂 보관함 / ⚙️ 설정

- 저장 버튼 따로 있음

## Claude API 프롬프트 설계

### 기본 프롬프트 구조
```
당신은 {업종} 사장님의 스레드 SNS 담당자입니다.
아래 조건에 맞는 스레드 게시글 3개를 작성해주세요.

[조건]
- 업종: {업종}
- 말투: {말투}
- 이모지: {이모지 옵션}
- 해시태그: {해시태그 옵션}
- 주제: {주제}
- 글자 수: 150자 내외

[출력 형식]
글1: ...
글2: ...
글3: ...
```

### 업종별 프롬프트 특성
- 카페: 음료/디저트/분위기/날씨 감성 강조
- 음식점: 오늘의 메뉴/재료/영업시간 중심
- 부동산: 시세/매물/생활정보 신뢰감 있게
- 헤어샵: 스타일링/예약/이벤트 친근하게

### 말투 옵션 프롬프트 반영
- 반말체: "친한 친구에게 말하듯 반말로"
- 존댓말체: "고객에게 정중하게 존댓말로"
- 친근한 말투: "동네 단골손님에게 말하듯 편안하게"
- 정보 전달형: "핵심 정보만 간결하게"
- 공감 유도형: "공감과 질문을 유도하는 방식으로"

### 이모지 옵션 프롬프트 반영
- 많이: "이모지를 문장마다 적극적으로 사용"
- 적당히: "이모지를 2~3개 적절히 사용"
- 안 씀: "이모지 없이 텍스트만"

### 해시태그 옵션 프롬프트 반영
- ON: "마지막 줄에 관련 해시태그 3~5개 추가"
- OFF: "해시태그 없이"

### AI 보완 옵션 프롬프트 처리
- 더 유머러스하게: "위 글을 더 유머러스하고 재치있게 수정해줘"
- 더 짧게: "위 글을 50자 이내로 핵심만 남기고 줄여줘"
- 더 길게: "위 글을 200자 이상으로 자세하게 늘려줘"
- 해시태그 추가: "위 글 마지막에 해시태그 3~5개 추가해줘"
- 톤 변경: "위 글을 {선택한 말투}로 바꿔줘"
- 다시 생성: 기본 프롬프트로 재호출

## 개발 규칙

### 공통 규칙
- 모든 코드는 TypeScript 사용
- Tailwind CSS만 사용 (별도 CSS 파일 금지)
- 모바일 우선 반응형 (모바일 → PC 순서로 작성)
- 컴포넌트 Props 타입 항상 정의

### 컴포넌트 규칙
- 재사용 컴포넌트는 components/ 폴더에 저장
- 로딩 상태 항상 포함
- 에러 상태 항상 포함
- 빈 상태(데이터 없음) UI 항상 포함

### API 규칙
- 모든 API 호출 try/catch 필수
- 로딩 / 에러 / 성공 상태 항상 관리
- 환경변수는 .env.local 에서만 관리
- API 키 코드에 직접 입력 금지
- 글 생성 버튼 디바운스 처리 필수 (중복 호출 방지)
- API Route에서 하루 생성 횟수 서버 사이드 체크 필수

### Supabase 규칙
- 모든 쿼리 에러 처리 필수
- 인증 상태 확인 후 데이터 접근
- RLS (Row Level Security) 항상 적용

### Supabase 플랜 보안 설계
클라이언트 우회 방지를 위해 프론트 + DB 이중 체크 구조 적용
```
사용자 액션
→ 1차: usePlan.ts 클라이언트 체크 (UX용)
→ 2차: API Route에서 플랜 재확인
→ 3차: Supabase RLS에서 DB 레벨 차단
```

#### DB 레벨 RLS 정책
- posts 테이블
  - 무료 플랜: 7일 이내 데이터만 SELECT 허용
  - 무료 플랜: 하루 1개 초과 INSERT 차단
  - 프로 플랜: 제한 없음
- users 테이블
  - 본인 데이터만 SELECT/UPDATE 허용
  - plan 컬럼은 서버에서만 UPDATE 허용 (클라이언트 직접 변경 불가)

#### API Route 플랜 체크 필수 항목
- 글 생성 API: 하루 생성 횟수 초과 여부 확인
- 보관함 API: 7일 이후 데이터 접근 차단
- 자동 생성 API: 프로 플랜 여부 확인
- AI 보완 API: 하루 10회 초과 여부 확인

### 플랜 제한 규칙
- 유료 기능 접근 시 플랜 체크 필수
- 무료 플랜 접근 시 UpgradeModal 컴포넌트 출력
- 플랜 체크 로직은 훅으로 분리 (hooks/usePlan.ts)

### 결제 관련 주의사항
- 결제 완료 후 Webhook으로 플랜 업데이트 처리
- 결제 실패 / 중복 결제 예외 처리 필수
- 구독 만료 시 플랜 자동 다운그레이드 처리 (Supabase Edge Functions)

### 프로젝트 구조
```
app/                  ← 페이지
components/           ← 공통 컴포넌트
hooks/                ← 커스텀 훅
lib/                  ← 유틸 함수 / Supabase 클라이언트
types/                ← TypeScript 타입 정의
```

### 주요 파일 경로
```

```

```
1단계: 프로젝트 세팅 + 환경변수 설정
2단계: 랜딩 페이지 UI
3단계: 카카오 로그인 + Supabase Auth 연동
4단계: 온보딩 페이지
5단계: 대시보드 (요일 탭 + 색상 구분)
6단계: 글 자동 생성 (Claude API 연동)
7단계: AI 보완 옵션
8단계: 보관함
9단계: 설정 페이지
10단계: 무료/유료 플랜 제한 + UpgradeModal
11단계: 구독 결제 연동 (토스페이먼츠)
12단계: 인스타그램 확장
13단계: Threads API 연동 (OAuth + 자동 업로드)
```

---

## 향후 확장 계획
- 프리랜서 개발자 타겟 확장
- 인스타그램 카드뉴스 자동 생성
- AI 이미지 자동 생성 추가
- Threads API 연동으로 완전 자동 업로드
- 구독료 인상 (이미지 생성 추가 시)
- 검색 기능 추가
- 이메일 발송 (결제 영수증 / 구독 만료 안내)

## Threads API 연동 설계 (13단계)

MVP는 수동 복사 업로드. 2단계에서 Threads API OAuth 연동.

### 환경변수 (Phase 13에 추가)
```
THREADS_APP_ID=
THREADS_APP_SECRET=
THREADS_REDIRECT_URI=
```

### users 테이블 추가 컬럼 (Phase 13)
```
threads_access_token (암호화 저장)
threads_user_id
threads_connected_at
threads_token_expires_at
```

### API Route (Phase 13)
```
app/api/threads/connect/route.ts   ← OAuth 연동
app/api/threads/publish/route.ts   ← 게시글 업로드
app/api/threads/status/route.ts    ← 업로드 상태 확인
```
